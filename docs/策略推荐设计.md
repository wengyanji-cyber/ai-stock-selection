# 策略推荐设计

## 1. 概述

### 1.1 目标
为用户智能推荐最适合的投资策略，提升投资决策效率和收益率。

### 1.2 核心原则
- **个性化**：基于用户画像精准匹配
- **动态化**：根据市场环境和用户行为实时调整
- **可解释**：每个推荐都有明确的理由
- **可反馈**：支持用户反馈优化推荐效果

## 4. 推荐算法

### 4.1 算法流程

```
输入: 用户ID
输出: 推荐策略列表(Top-N)

步骤:
1. 获取用户画像 P
2. 获取活跃策略列表 S = [s1, s2, ..., sn]
3. 获取市场环境 M
4. 对每个策略 si ∈ S:
   a. 计算用户匹配度: userScore = calcUserMatch(si, P)
   b. 计算市场适配度: marketScore = calcMarketFit(si, M)
   c. 计算策略质量分: qualityScore = calcQuality(si)
   d. 综合评分: totalScore = w1*userScore + w2*marketScore + w3*qualityScore
5. 按 totalScore 降序排序
6. 取 Top-N 作为推荐结果
7. 生成推荐理由
8. 返回推荐列表
```

### 4.2 用户匹配度计算

```typescript
function calcUserMatch(strategy: Strategy, user: UserProfile): number {
  let score = 0
  
  // 1. 风险偏好匹配 (权重: 35%)
  const riskMatch = calcRiskMatch(strategy.riskLevel, user.riskType)
  score += riskMatch * 0.35
  
  // 2. 收益期望匹配 (权重: 25%)
  const returnMatch = calcReturnMatch(strategy.expectedReturn, user.expectedReturn)
  score += returnMatch * 0.25
  
  // 3. 持仓周期匹配 (权重: 20%)
  const periodMatch = calcPeriodMatch(strategy.holdingPeriod, user.avgHoldingPeriod)
  score += periodMatch * 0.20
  
  // 4. 板块偏好匹配 (权重: 15%)
  const sectorMatch = calcSectorMatch(strategy.suitableSectors, user.preferredSectors)
  score += sectorMatch * 0.15
  
  // 5. 资金门槛检查 (权重: 5%)
  const capitalMatch = user.currentCapital >= strategy.minCapital ? 1 : 0.5
  score += capitalMatch * 0.05
  
  return Math.min(score * 100, 100)  // 归一化到0-100
}

// 风险偏好匹配
function calcRiskMatch(strategyRisk: string, userRisk: string): number {
  const riskMap = { 'low': 1, 'medium': 2, 'high': 3 }
  const diff = Math.abs(riskMap[strategyRisk] - riskMap[userRisk])
  return Math.max(0, 1 - diff * 0.4)  // 差异越大分数越低
}

// 收益期望匹配
function calcReturnMatch(strategyReturn: Range, userExpectation: number): number {
  const avgReturn = (strategyReturn.min + strategyReturn.max) / 2
  const diff = Math.abs(avgReturn - userExpectation)
  return Math.max(0, 1 - diff / userExpectation)
}

// 持仓周期匹配
function calcPeriodMatch(strategyPeriod: Range, userPeriod: number): number {
  const avgPeriod = (strategyPeriod.min + strategyPeriod.max) / 2
  const diff = Math.abs(avgPeriod - userPeriod)
  return Math.max(0, 1 - diff / Math.max(avgPeriod, userPeriod))
}

// 板块偏好匹配
function calcSectorMatch(strategySectors: string[], userSectors: string[]): number {
  const intersection = strategySectors.filter(s => userSectors.includes(s))
  return intersection.length / Math.max(strategySectors.length, userSectors.length)
}
```

### 4.3 市场适配度计算

```typescript
function calcMarketFit(strategy: Strategy, market: MarketEnvironment): number {
  let score = 0
  
  // 1. 市场趋势适配 (权重: 40%)
  if (strategy.suitableMarket === 'all' || strategy.suitableMarket === market.indexTrend) {
    score += 0.40
  } else if (
    (strategy.suitableMarket === 'bull' && market.indexTrend === 'neutral') ||
    (strategy.suitableMarket === 'bear' && market.indexTrend === 'neutral')
  ) {
    score += 0.20
  }
  
  // 2. 波动率适配 (权重: 30%)
  const volatilityFit = 1 - Math.abs(strategy.volatility - market.marketVolatility) / 0.5
  score += Math.max(0, volatilityFit) * 0.30
  
  // 3. 板块热度适配 (权重: 30%)
  const sectorHeatScore = strategy.suitableSectors.reduce((sum, sector) => {
    return sum + (market.sectorHeat.get(sector) || 0)
  }, 0) / strategy.suitableSectors  score += sectorHeatScore * 0.30
  
  return Math.min(score * 100, 100)
}

// 策略质量评分
function calcQuality(strategy: Strategy): number {
  const backtest = strategy.backtestResult
  
  // 综合质量分 = 胜率*0.3 + 收益分*0.3 + 风险调整收益*0.4
  const winRateScore = backtest.winRate
  const returnScore = Math.min(backtest.avgReturn / 30, 1)  // 30%收益为满分
  const riskAdjScore = (backtest.sharpeRatio + 1) / 3       // 夏普比率归一化
  
  return (winRateScore * 0.3 + returnScore * 0.3 + riskAdjScore * 0.4) * 100
}

## 5. 推荐理由生成

### 5.1 规则模板

```typescript
function generateReason(strategy: Strategy, user: UserProfile, matchScore: number): string {
  const reasons: string[] = []
  
  // 风险偏好匹配
  if (strategy.riskLevel === user.riskType) {
    reasons.push(`该策略的${strategy.riskLevel}风险等级与您的偏好完全匹配`)
  }
  
  // 收益期望匹配
  const avgReturn = (strategy.expectedReturn.min + strategy.expectedReturn.max) / 2
  if (Math.abs(avgReturn - user.expectedReturn) < 5) {
    reasons.push(`预期收益${avgReturn}%符合您的收益目标`)
  }
  
  // 板块偏好匹配
  const commonSectors = strategy.suitableSectors.filter(s => 
    user.preferredSectors.includes(s)
  )
  if (commonSectors.length > 0) {
    reasons.push(`策略关注的${commonSectors.join('、')}板块正是您感兴趣的领域`)
  }
  
  // 市场环境适配
  reasons.push(`当前市场环境适合该策略发挥`)
  
  // 历史表现
  if (strategy.backtestResult.winRate > 0.6) {
    reasons.push(`历史胜率${(strategy.backtestResult.winRate * 100).toFixed(0)}%表现优秀`)
  }
  
  return reasons.join('，')
}
```

## 6. 应用场景

### 6.1 新用户首次登录
- **场景**：用户完成注册，首次进入策略推荐页
- **策略**：基于问卷结果推荐最匹配的策略
- **展示**：Top-3策略，突出最佳匹配

### 6.2 每日市场开盘前
- **场景**：每日9:00推送当日策略建议
- **策略**：结合隔夜市场数据调整推荐排序
- **展示**：推送通知 + 应用内醒目位置

### 6.3 用户持仓亏损时
- **场景**：用户某只股票亏损超过5%
- **策略**：推荐止损策略或防御型策略
- **展示**：弹窗提醒 + 策略切换建议

### 6.4 市场环境剧变时
- **场景**：大盘单日涨跌超过3%
- **策略**：重新计算所有策略适配度，推荐应对策略
- **展示**：紧急通知 + 策略调整建议

## 7. 系统实现

### 7.1 技术栈
- **后端**：Node.js + TypeScript + Fastify
- **数据库**：MySQL（用户画像）+ Redis（缓存）
- **算法**：Python（复杂模型）/ TypeScript（简单规则）
- **机器学习**：TensorFlow.js / scikit-learn（可选）

### 7.2 数据流
```
用户行为 → Kafka → 实时计算 → 更新画像 → 触发重算 → 推送新推荐
```

### 7.3 性能优化
- **缓存策略**：用户推荐结果缓存5分钟
- **预计算**：每日凌晨预计算所有用户推荐
- **增量更新**：仅变化时重新计算

### 7.4 A/B测试
- 对照组：随机推荐
- 实验组：算法推荐
- 指标：点击率、采纳率、实际收益率

## 10. 风险控制

### 10.1 避免过度拟合
- 定期用新数据验证模型
- 控制模型复杂度
- 保留随机推荐作为对照

### 10.2 合规要求
- 推荐必须附带风险提示
- 不得承诺收益
- 保留用户自主决策权

### 10.3 异常处理
- 数据缺失时使用默认值
- 算法异常时回退到规则推荐
- 监控报警机制
