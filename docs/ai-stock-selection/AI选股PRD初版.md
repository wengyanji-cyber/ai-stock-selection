# AI选股PRD初版

## 1. 文档信息

产品名称：AI短线投研助手

文档版本：V1.0

文档目标：定义首版 AI 选股产品的用户、场景、功能、数据要求、页面结构和验收标准，用于产品、设计、研发和运营协同。

## 2. 项目背景

当前普通投资者在短线交易中普遍面临信息过载、筛选效率低、缺乏结构化结论、容易情绪化交易等问题。市场已有工具多偏向原始数据展示，缺乏对热点板块、候选股和风险点的结构化输出。

因此，首版产品目标不是替用户交易，而是帮助用户在更短时间内完成以下动作：

1. 看清今天重点方向。
2. 快速筛出少量候选股票。
3. 判断个股当前状态。
4. 形成可执行的观察计划。
5. 持续跟踪自选股变化。

## 3. 产品目标

首版产品目标包括两个层面。

业务目标：

1. 验证用户是否愿意每天打开产品。
2. 验证用户是否愿意使用个股诊断和候选池。
3. 验证试用后是否存在真实付费意愿。

产品目标：

1. 输出少量高质量的候选股票。
2. 提供散户能看懂的结论和风险提示。
3. 形成盘前、盘中、盘后的基础使用闭环。

## 4. 目标用户

首阶段目标用户为半懂技术面的活跃散户。

用户特征：

1. 有看盘习惯。
2. 有自选股列表。
3. 懂一点均线、支撑、压力、量能。
4. 喜欢热点、题材、龙头。
5. 愿意为效率和结构化结论付费。

## 5. 使用场景

### 5.1 盘前场景

用户想在开盘前快速知道：

1. 今天市场偏强还是偏弱。
2. 哪些板块值得重点看。
3. 今天最值得跟踪的少量股票有哪些。

### 5.2 盘中场景

用户想知道：

1. 某只股票当前是否还值得看。
2. 热点是否转弱。
3. 自己关注的股票状态是否变化。

### 5.3 盘后场景

用户想复盘：

1. 今天哪些判断有效。
2. 哪些风险提示兑现了。
3. 明天应该继续盯哪些方向。

## 6. 产品范围

首版仅覆盖：

1. A 股。
2. 两个热点行业，建议 AI 算力链和机器人/智能制造。
3. 3 到 5 个交易日短周期观察。
4. 日频或准实时更新。

首版明确不做：

1. 港股和美股覆盖。
2. 自动交易。
3. 代客操盘。
4. 券商账户直连下单。
5. 社区发帖和复杂社交功能。

## 7. 核心功能

### 7.1 首页

目标：在最短时间内回答用户三个问题。

1. 今天看什么。
2. 为什么看。
3. 今天的主要风险是什么。

功能模块：

1. 盘前一句话。
2. 今日重点板块。
3. 今日重点候选。
4. 风险提醒。

### 7.2 候选池页

目标：展示每日候选股池，让用户系统查看重点候选与扩展观察股。

功能模块：

1. 候选股列表。
2. 板块筛选。
3. 风险等级筛选。
4. 候选详情展开。

单只候选股字段：

1. 股票名称。
2. 股票代码。
3. 所属板块。
4. 一句话结论。
5. 关注理由。
6. 观察区间。
7. 支撑位。
8. 压力位。
9. 止损位。
10. 失效条件。
11. 风险标签。

### 7.3 个股诊断页

目标：让用户输入股票代码后快速得到结构化判断。

输入：股票代码或名称。

输出内容：

1. 一句话判断。
2. 趋势状态。
3. 强弱评级。
4. 所属板块。
5. 观察区间。
6. 支撑位。
7. 压力位。
8. 止损位。
9. 主要理由。
10. 风险提示。
11. 操作建议。

### 7.4 自选观察页

目标：围绕用户添加的股票形成高频回访理由。

功能模块：

1. 自选股列表。
2. 状态变化展示。
3. 转强、转弱、跌破关键位、板块退潮等标签。
4. 最新建议与风险提示。

### 7.5 每日复盘页

目标：增强用户对系统输出的理解与信任。

功能模块：

1. 市场回顾。
2. 热点板块表现。
3. 候选股表现。
4. 风险提示兑现情况。
5. 明日关注方向。

## 8. 内容输出规范

所有重点股票建议统一输出以下内容：

1. 一句话结论。
2. 1 到 3 条关注理由。
3. 观察区间。
4. 支撑位。
5. 压力位。
6. 止损位。
7. 失效条件。
8. 风险标签。

内容规范要求：

1. 先结论，后理由。
2. 语言必须通俗。
3. 风险提示必须和机会并列。
4. 避免使用收益承诺和绝对词。

## 9. 数据需求

首版所需数据分为三层。

必须数据：

1. A 股基础行情。
2. 分时和日线数据。
3. 成交额、换手率、振幅、涨跌幅。
4. 行业和概念板块映射。
5. 公告与停牌信息。
6. 风险标签数据。

增强数据：

1. 新闻流。
2. 板块热度。
3. 舆情热度。
4. 板块资金强弱。

后置数据：

1. 工商信息。
2. 专利数据。
3. 科研数据。
4. 港股美股联动数据。

## 10. 核心逻辑

首版产品建议采用三层结构：

1. 特征与规则层，负责板块热度、趋势、量价、风险过滤。
2. 排序层，负责生成重点候选和扩展观察列表。
3. AI 解释层，负责把结果翻译成用户能看懂的结论。

当前阶段不追求复杂黑盒预测模型，优先追求稳定性、可解释性和可持续迭代。

## 11. 页面流程

用户核心使用路径建议如下：

1. 打开首页，看盘前一句话和重点板块。
2. 进入候选池，查看今日重点股。
3. 点击单只股票查看详细计划。
4. 输入自选股做个股诊断。
5. 把感兴趣的股票加入自选观察。
6. 盘后查看复盘和明日关注方向。

## 12. 非功能要求

### 12.1 性能要求

1. 首页加载速度应可接受。
2. 个股诊断响应应尽量控制在短时间内。
3. 候选池数据更新应保持稳定。

### 12.2 可用性要求

1. 页面结构清晰。
2. 重点内容优先展示。
3. 点位信息易读。
4. 移动端和桌面端都能正常浏览。

### 12.3 合规要求

1. 不承诺收益。
2. 不代客操作。
3. 所有页面保留辅助决策性质说明。
4. 禁止出现高胜率暴利类话术。

## 13. MVP验收标准

功能验收标准：

1. 首页、候选池页、个股诊断页、自选观察页、复盘页可正常访问。
2. 每日可生成固定数量候选股。
3. 个股诊断可返回完整结构化内容。
4. 用户可将股票加入观察列表。
5. 风险标签与点位字段可正确展示。

业务验收标准：

1. 种子用户愿意连续使用。
2. 候选池有实际点击和查看。
3. 个股诊断有明确使用频次。
4. 用户反馈认为内容看得懂、有帮助。
5. 出现初步试用转付费意愿。

## 14. 埋点建议

建议从首版开始埋以下行为数据：

1. 首页访问次数。
2. 盘前一句话曝光次数。
3. 候选池点击率。
4. 单只候选股展开率。
5. 个股诊断发起次数。
6. 自选股添加次数。
7. 复盘页查看次数。
8. 试用转付费点击率。

## 15. 风险与限制

当前版本的主要风险包括：

1. 候选质量不稳定。
2. 热点识别在极端行情下失效。
3. 用户对短线结果预期过高。
4. 运营或销售表述越界引发合规风险。

因此首版必须坚持两个原则：

1. 聚焦小范围功能，不做大而全。
2. 聚焦辅助与解释，不越界到代操与承诺收益。

## 16. 后续版本方向

如果首版验证通过，可考虑后续迭代：

1. 午后补充更新。
2. 板块龙头联动图谱。
3. 更细的风险分层。
4. 更强的个股解释能力。
5. 与持牌机构的能力合作接口。

## 17. 结语

首版产品的成功，不在于一开始做成全能交易终端，而在于先证明这件事：

普通投资者愿意每天打开一个能给出少量高质量结论、讲清理由和风险的 AI 投研辅助产品。
