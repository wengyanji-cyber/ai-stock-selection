# 8周MVP功能清单 + 数据源 + 指标表

## 1. MVP目标

在 8 周内做出一款可供小范围用户试用的 A 股 AI 短周期投研辅助产品，完成以下六件事：

1. 两个行业的热点追踪。
2. 每日候选池输出。
3. 个股诊断。
4. 风险提示。
5. 连续 4 周纸面模拟跟踪。
6. 小规模种子用户试用。

## 2. MVP范围

只做：

1. A 股。
2. 2 个热点行业。
3. 日频或准实时更新。
4. 观察周期 3 到 5 个交易日。
5. 人工可验证的候选与理由输出。

不做：

1. 港股。
2. 美股。
3. 自动交易。
4. 代客操盘。
5. 高频毫秒级策略。
6. 太多行业同时覆盖。

## 3. 建议首批行业

1. AI 算力链。
2. 机器人/智能制造。

选择原因：

1. 热点属性强。
2. 新闻驱动明显。
3. 散户认知度高。
4. 板块联动较清楚。
5. 适合短线观察。

## 4. 第1周：明确边界与成功标准

目标：把做什么、不做什么、怎么衡量成功一次定清楚。

输出物：

1. 用户画像定义。
2. MVP 功能清单。
3. 候选股输出模板。
4. 指标口径定义。
5. 合规表述边界。

具体任务：

1. 明确目标用户为半懂技术面的活跃散户。
2. 明确只做 A 股两大行业。
3. 定义每日候选池数量，建议 3 只重点候选 + 5 只扩展观察。
4. 定义单只股票输出字段：股票名称、所属板块、关注理由、观察区间、支撑、压力、止损、失效条件、风险标签。
5. 定义评估口径：5 日胜率、5 日超额收益、最大回撤、命中热点比例。

## 5. 第2周：数据接入与特征框架

目标：先打通够用的数据，不追求豪华。

需要的数据源类型：

1. 行情数据：日线、分钟线、成交额、换手率、振幅、涨跌幅、封板信息。
2. 板块数据：行业板块、概念板块、板块涨跌幅、板块热度变化、龙头联动。
3. 公告和新闻：财报预警、减持、异动、监管、停牌、重大事项。
4. 热度数据：搜索热度、讨论热度、新闻频次、社媒提及强度。

建议前期优先级：

1. 必须有：行情、板块、公告。
2. 最好有：新闻、基础热度。
3. 可后置：工商、专利、科研。

第一版特征建议：

1. 趋势特征：近 5 日、10 日、20 日涨跌幅，均线关系，是否突破前高。
2. 量价特征：成交额变化、换手率变化、量能放大程度。
3. 波动特征：振幅、回撤、涨停/大涨频次。
4. 热点特征：所属板块强度、板块排名、板块持续性。
5. 风险特征：是否 ST、是否高风险公告、是否极低流动性。

## 6. 第3周：规则引擎与初筛系统

目标：先用规则和过滤器做出看起来靠谱的候选股池，而不是急着上复杂模型。

第一版过滤规则建议：

1. 排除 ST、退市风险、高监管风险标的。
2. 排除流动性太差的股票。
3. 排除无量上涨或极端异常标的。
4. 只保留目标行业和热点关联股票。
5. 优先保留板块联动明显的股票。

第一版机会信号建议：

1. 板块走强 + 个股放量突破。
2. 板块持续热度 + 个股换手活跃。
3. 龙头上涨带动补涨股启动。
4. 回调后量缩企稳再转强。
5. 公告/事件催化叠加板块共振。

## 7. 第4周：排序模型与候选池生成

目标：把初筛标的按值得看的程度排出来。

第一版打分结构建议：

1. 机会分：趋势、放量、板块联动、突破状态。
2. 热度分：新闻、热度、题材活跃度、市场关注度。
3. 稳定性分：趋势连续性、回撤可控性、量价协调性。
4. 风险分：公告风险、监管风险、波动失控、流动性问题。

总分逻辑：

总分 = 机会分 + 热度分 + 稳定性分 - 风险扣分

然后每日输出：

1. 重点候选 3 只。
2. 扩展观察 5 只。
3. 重点板块 2 到 3 个。

注意：不要每天十几二十只，太多就失去产品价值。

## 8. 第5周：AI解释层与内容模板

目标：把机器结果翻译成散户能看懂的语言。

每只股票建议固定输出六个字段：

1. 一句话结论。
2. 关注理由。
3. 观察区间。
4. 压力位与支撑位。
5. 止损或失效条件。
6. 风险提示。

示例：

结论：板块热度仍在，个股量价结构偏强，可关注回踩支撑后的承接表现。

理由：所属板块近两日持续走强；个股放量突破阶段高点；换手活跃但未出现明显放量滞涨。

观察区间：XX 至 XX

压力位：XX

止损位：XX

失效条件：跌破支撑且板块同步转弱。

风险提示：短期涨幅较大，若板块分歧扩大，波动会明显加剧。

## 9. 第6周：前端页面与基础交互

目标：把最小可用界面做出来，不追求复杂图表终端。

建议页面只有四个：

1. 首页：盘前一句话、重点板块、重点候选。
2. 候选池页：展示每日重点股和扩展观察股。
3. 个股诊断页：输入代码返回诊断结果。
4. 自选/持仓观察页：展示用户关注股票的状态变化。

首页必须突出三件事：

1. 今天看什么。
2. 为什么看。
3. 哪些不能碰。

## 10. 第7周：纸面模拟运行

目标：开始做真正的连续验证，禁止回看后改答案。

执行方式：

1. 每天固定两个时间点更新，比如盘前一次、午后一次。
2. 每次输出固定数量候选，不临时加塞，不事后解释。
3. 每只股票记录入选时间、理由、观察区间、失效条件、5 日表现。
4. 连续记录至少 15 到 20 个交易日。

这一阶段要看：

1. 候选是否稳定。
2. 理由是否一致。
3. 热点命中率如何。
4. 失败样本有没有共性。

## 11. 第8周：小范围试用与留存验证

目标：不只看模型，也看用户行为。

试用建议：

1. 邀请 20 到 50 个种子用户。
2. 连续试用 10 到 14 天。
3. 每天都能看到盘前结论与候选池。
4. 开放个股诊断和自选观察。

核心收集指标：

1. 次日留存。
2. 7 日留存。
3. 人均每天打开次数。
4. 人均诊股次数。
5. 候选池点击率。
6. 自选股添加率。
7. 试用结束后转付费意愿。

如果用户每天都打开，但收益指标一般，说明产品的陪伴价值成立。
如果收益看起来不错，但用户不爱打开，说明产品表达和交互有问题。

## 12. MVP功能清单

必须做：

1. 每日盘前一句话。
2. 热点板块榜。
3. 每日候选池。
4. 个股极简诊断。
5. 自选股观察。
6. 风险标签。
7. 候选池历史记录。
8. 数据看板基础统计。

最好做：

1. 午后补充更新。
2. 板块龙头联动关系。
3. AI 问答入口。
4. 用户反馈按钮。
5. 模拟跟踪结果页。

先不做：

1. 自动交易。
2. 港美股同步。
3. 社区发帖。
4. 高级回测终端。
5. 复杂自定义策略编辑器。
6. 太重的可视化行情系统。

## 13. MVP所需数据源表

建议按必须、最好、后置分层。

必须：

1. A 股基础行情。
2. 日线与分钟线。
3. 成交额、换手率、振幅。
4. 涨跌停数据。
5. 行业/概念板块映射。
6. 公告与停牌信息。
7. 风险标签数据。

最好：

1. 新闻流。
2. 舆情热度。
3. 板块资金强弱。
4. 异动监控。
5. 龙虎榜类热度参考。

后置：

1. 工商信息。
2. 专利数据库。
3. 科研论文数据。
4. 复杂另类数据。
5. 港股美股联动数据。

## 14. 评估指标表

不要只看单一准确率，建议用以下指标。

策略效果指标：

1. 5 日胜率：候选股在观察期内正收益的比例。
2. 5 日平均收益：候选股平均涨跌幅。
3. 5 日超额收益：相对行业指数或宽基指数的超额表现。
4. 最大回撤：观察期内最差回撤。
5. 热点命中率：候选股是否命中当期主流热点。
6. 失效触发率：多少候选股在失效条件前先走坏。

产品使用指标：

1. 日活。
2. 次日留存。
3. 7 日留存。
4. 诊股使用频次。
5. 候选股点击率。
6. 自选股添加率。
7. 试用转付费率。

内容解释指标：

1. 用户是否看懂。
2. 用户是否觉得理由可信。
3. 用户是否愿意按计划执行。
4. 用户是否觉得风险提示有帮助。

## 15. MVP成功标准

8 周结束后，如果满足下面任意四条，就值得继续投：

1. 种子用户 7 日留存达到可接受水平。
2. 候选池有持续点击和查看。
3. 用户愿意频繁使用个股诊断。
4. 重点候选在样本外测试中有正向超额。
5. 用户反馈认为节省时间、看得懂。
6. 试用用户中出现明确付费意愿。

如果只满足模型看着还行，但没有留存和转化信号，不建议继续重投。

## 16. 下一步应补齐的三份文件

为了把项目从想法推进到执行，建议继续补三份文档：

3. 合规与对外表述文档：写清楚哪些词禁用，哪些服务暂不做，哪些收费方式不能碰。
