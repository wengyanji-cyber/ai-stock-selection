# AI选股数据与模型设计文档

## 1. 文档目的

本文件用于定义 AI 选股项目首版的数据范围、特征体系、规则逻辑、排序模型、解释层设计和评估框架，帮助产品、数据、后端团队统一理解系统的技术边界。

首版核心原则：

1. 先求稳定可解释。
2. 先求小范围可验证。
3. 先求可持续迭代，不追求一开始做成复杂黑盒。

## 2. 目标定义

首版系统的目标不是预测所有股票会不会涨，而是完成三件事：

1. 从热点行业中筛出更值得关注的少量候选股。
2. 为每只股票生成结构化的观察计划与风险提示。
3. 为用户提供可理解、可追踪、可复盘的结论。

因此，系统本质上是一个候选筛选与排序系统，而不是纯预测系统。

## 3. 首版建模范围

首版只覆盖：

1. A 股。
2. 两个热点行业，建议 AI 算力链和机器人/智能制造。
3. 观察周期 3 到 5 个交易日。
4. 日频为主，必要时补充午后更新。

首版不覆盖：

1. 港股和美股。
2. 自动交易。
3. 高频 tick 级策略。
4. 跨市场套利。

## 4. 数据分层设计

建议按三层组织数据。

### 4.1 基础市场数据

这是首版必须打通的数据层。

包括：

1. 股票基本信息。
2. 日线行情。
3. 分钟级行情。
4. 成交额。
5. 换手率。
6. 振幅。
7. 涨跌幅。
8. 涨停、跌停、封板状态。

### 4.2 板块与事件数据

这是判断热点与联动关系的关键层。

包括：

1. 行业板块映射。
2. 概念板块映射。
3. 板块涨跌表现。
4. 板块热度变化。
5. 龙头股候选。
6. 公告。
7. 停牌。
8. 异动。
9. 监管和风险事件。

### 4.3 热度与增强数据

这是帮助排序和解释增强的层。

包括：

1. 新闻频次。
2. 关键词热度。
3. 舆情提及强度。
4. 板块关注度变化。
5. 龙虎榜类热度参考。

## 5. 数据优先级

为了避免前期系统过重，建议按优先级推进。

必须先做：

1. 行情数据。
2. 板块数据。
3. 公告与风险数据。

第二优先级：

1. 新闻流。
2. 舆情热度。
3. 板块资金热度。

后置：

1. 工商信息。
2. 专利数据。
3. 科研数据。
4. 复杂另类数据。

## 6. 特征体系设计

首版特征建议分为五大类。

### 6.1 趋势特征

用于判断个股是否处于可观察阶段。

建议特征包括：

1. 近 3 日涨跌幅。
2. 近 5 日涨跌幅。
3. 近 10 日涨跌幅。
4. 均线位置关系。
5. 是否突破阶段高点。
6. 是否回踩后重新企稳。

### 6.2 量价特征

用于判断上涨或反弹是否有资金承接。

建议特征包括：

1. 成交额变化率。
2. 换手率变化率。
3. 放量程度。
4. 缩量回踩特征。
5. 量价是否协同。

### 6.3 波动特征

用于判断股票是否属于可控波动而非混乱波动。

建议特征包括：

1. 近 5 日平均振幅。
2. 单日极端波动次数。
3. 最大回撤。
4. 涨停或大涨频次。

### 6.4 热点特征

用于判断股票是否处于主流热点或跟随热点。

建议特征包括：

1. 所属板块排名。
2. 板块近 3 日强度。
3. 板块持续性。
4. 个股与板块同步度。
5. 是否为板块核心或补涨股。

### 6.5 风险特征

用于过滤不适合首版短周期观察的标的。

建议特征包括：

1. 是否 ST。
2. 是否存在重大风险公告。
3. 是否流动性过低。
4. 是否短期过度异动。
5. 是否存在明显监管风险。

## 7. 首版规则引擎

首版不要把全部希望压在模型上，先建立规则引擎做第一层筛选。

规则引擎作用：

1. 过滤明显不合适的股票。
2. 保证候选池风格一致。
3. 降低垃圾样本进入排序系统的概率。

建议规则包括：

1. 排除 ST 和退市风险标的。
2. 排除流动性明显不足的股票。
3. 排除高风险公告影响期内标的。
4. 排除偏离目标热点过远的股票。
5. 优先保留板块联动强、量价结构清晰的股票。

## 8. 候选生成逻辑

建议用两阶段生成候选。

### 8.1 第一阶段：初筛

从目标板块中筛出满足基础条件的股票。

例如：

1. 板块热度位于前列。
2. 个股近期走势未明显走坏。
3. 量价结构至少满足最低标准。
4. 无明显重大风险事件。

### 8.2 第二阶段：排序

在初筛结果中按综合得分做排序，输出：

1. 重点候选 3 只。
2. 扩展观察 5 只。

## 9. 排序模型设计

首版建议采用可解释的综合打分结构，而不是复杂黑盒。

建议分成四个子分：

1. 机会分。
2. 热度分。
3. 稳定性分。
4. 风险扣分。

参考公式：

总分 = 机会分 + 热度分 + 稳定性分 - 风险扣分

### 9.1 机会分

反映当前结构是否具备观察价值。

可由以下部分组成：

1. 趋势状态。
2. 放量突破。
3. 回踩企稳。
4. 板块共振。

### 9.2 热度分

反映当前是否具备市场关注度。

可由以下部分组成：

1. 新闻热度。
2. 板块热度。
3. 题材关注度。
4. 龙头带动效应。

### 9.3 稳定性分

反映这只股票是否更适合普通投资者观察。

可由以下部分组成：

1. 回撤是否可控。
2. 量价是否协调。
3. 波动是否过度失控。
4. 结构是否清晰。

### 9.4 风险扣分

反映需要主动降低优先级的因素。

可由以下部分组成：

1. 公告风险。
2. 监管风险。
3. 过高波动。
4. 流动性问题。

## 10. AI解释层设计

大模型在首版中的职责不是直接预测涨跌，而是负责解释和交互。

主要作用：

1. 把模型和规则结果翻译成自然语言。
2. 生成一句话结论。
3. 生成 1 到 3 条核心理由。
4. 生成风险提示和失效条件。
5. 响应用户的个股追问。

标准输出字段建议：

1. 一句话结论。
2. 关注理由。
3. 观察区间。
4. 支撑位。
5. 压力位。
6. 止损位。
7. 失效条件。
8. 风险标签。

## 11. 观察区间与点位生成思路

点位不建议完全由大模型自由生成，应优先来自规则或指标计算结果。

建议流程：

1. 由系统先算出观察区间、支撑、压力、止损候选值。
2. 再由解释层将点位转换为用户能理解的语言。

这样能降低大模型随意发挥带来的不稳定问题。

## 12. 评估框架

评估不能只看单一准确率，建议至少看三层指标。

### 12.1 候选效果指标

1. 5 日胜率。
2. 5 日平均收益。
3. 5 日超额收益。
4. 最大回撤。
5. 热点命中率。

### 12.2 失效指标

1. 触发止损比例。
2. 跌破支撑比例。
3. 入选后快速转弱比例。

### 12.3 用户理解指标

1. 用户是否看懂理由。
2. 用户是否理解风险提示。
3. 用户是否愿意继续查看。

## 13. 验证路径

首版建议按四步验证：

1. 历史回放。
2. 样本外滚动验证。
3. 连续纸面模拟。
4. 小范围种子用户试用。

注意事项：

1. 不能只看回测。
2. 不能频繁改口径导致结果失真。
3. 纸面模拟必须固定时间出结果，不能盘后改答案。

## 14. 数据表建议

首版可考虑以下核心数据表：

1. 股票基础信息表。
2. 行情日线表。
3. 行情分钟线表。
4. 板块映射表。
5. 板块热度快照表。
6. 公告事件表。
7. 风险标签表。
8. 候选生成结果表。
9. 个股解释结果表。
10. 模拟跟踪结果表。

## 15. 技术原则

首版建议遵循以下技术原则：

1. 规则优先于复杂建模。
2. 数据质量优先于数据数量。
3. 可解释优先于黑盒高复杂度。
4. 小范围稳定优先于全市场铺开。
5. 先构建复盘和记录能力，再追求更复杂信号。

## 16. 后续升级方向

如果首版验证成立，可逐步补充：

1. 板块轮动图谱。
2. 板块核心与补涨识别。
3. 午后动态更新能力。
4. 更细粒度的风险分层。
5. 更强的用户问答上下文能力。

## 17. 结论

首版技术路线应围绕一个目标展开：

用有限的数据、清晰的规则、可解释的排序和稳定的内容输出，做出一个普通投资者愿意每天使用的 AI 短周期投研辅助系统。
